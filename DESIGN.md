# DESIGN.md

이 문서는 그룹웨어 MVP1의 비주얼 디자인 시스템을 정의한다. 화면을 새로 만들거나 기존 화면을 수정할 때 여기 정의된 토큰과 패턴을 따른다. 색상 값 자체는 `Tailwind CSS v4`의 `@theme` 블록에 구현되며(`references/tailwind-v4-llms.txt` 참고), 이 문서는 그 값들을 왜, 어떻게 쓰는지에 대한 상위 기준이다.

---

## 1. 디자인 원칙

이 서비스는 마케팅 사이트가 아니라 **매일 업무 시간에 쓰는 사내 도구**다. 원칙은 세 가지다:

1. **명확성이 개성보다 우선한다.** 일정, 회의실 예약 같은 정보는 빠르게 읽고 다음 행동으로 넘어가야 한다. 장식적 요소(과도한 그림자, 애니메이션, 그라디언트)는 정보 인식을 방해하면 뺀다.
2. **밀도와 여백의 균형.** 캘린더, 회의실 현황판처럼 표 형태 정보가 많은 화면은 밀도를 높이되, 클릭 가능한 요소(버튼, 일정 블록)는 터치/클릭하기 충분한 크기를 유지한다(최소 32px 높이 권장).
3. **반복되는 패턴은 반드시 하나의 컴포넌트로.** 모달, 우측 배너, 상태 배지처럼 여러 화면에서 같은 형태가 쓰이는 경우, 화면마다 새로 만들지 않고 `shared/components/`의 공통 컴포넌트를 재사용한다(FRONTEND.md와 연결).

## 2. 컬러 시스템

화면/기능 설계서(1-9. 대표색상)에서 확정된 2가지 테마를 기준으로, 각 테마 안에서 쓰이는 역할별 토큰을 정의한다. 실제 CSS 구현은 `references/tailwind-v4-llms.txt`의 "런타임 테마 전환" 방식(방식 A, `data-theme` 속성)을 따른다.

### 테마 1 — 하늘색 (기본 테마)

| 역할 | 값 | 용도 |
|---|---|---|
| `--color-bg` | `#EDF1F6` | 페이지 배경 |
| `--color-surface` | `#FFFFFF` | 카드, 모달, 배너 등 콘텐츠 표면 |
| `--color-primary` | `#7c96c0` | 주요 버튼, 강조 링크, 선택된 탭 |
| `--color-text` | `#1F2937` | 본문 텍스트 |
| `--color-text-muted` | `#6B7280` | 보조 텍스트, 캡션 |
| `--color-border` | `#D9E2EC` | 구분선, 입력창 테두리 |

### 테마 2 — 보라색

| 역할 | 값 | 용도 |
|---|---|---|
| `--color-bg` | `#e2e1ee` | 페이지 배경 |
| `--color-surface` | `#FFFFFF` | 카드, 모달, 배너 등 콘텐츠 표면 |
| `--color-primary` | `#5541A4` | 주요 버튼, 강조 링크, 선택된 탭 |
| `--color-text` | `#292149` | 배경 위 본문 텍스트 |
| `--color-text-muted` | `#736e90` | 보조 텍스트 (연보라, 화면설계서의 "연보라"에 대응) |
| `--color-border` | `#7B68C4` | 구분선 |


### 공통 시맨틱 컬러 (테마와 무관하게 고정)

| 역할 | 값 | 용도 |
|---|---|---|
| `--color-danger` | `#DC2626` | 삭제, 예약 취소, 에러 메시지 |
| `--color-success` | `#16A34A` | 예약 완료, 저장 성공 |
| `--color-warning` | `#D97706` | 시간 겹침 경고 등 |

에러/성공 색상은 두 테마 모두에서 동일하게 유지해서, 사용자가 테마를 바꿔도 "위험한 동작"에 대한 인지가 흔들리지 않게 한다.

### 일정 색상 라벨 (`schedules.color_label`)

`schedule.md`에서 확정한 `color_label` 컬럼에 저장할 프리셋 팔레트. 임의의 색을 고르게 하지 않고, 아래 6개 중에서 선택하게 한다(캘린더 가독성 유지, 접근성상 서로 구분 가능한 색만 허용):

```
빨강 #EF4444 · 주황 #F97316 · 노랑 #EAB308 · 초록 #22C55E · 파랑 #3B82F6 · 보라 #A855F7
```

## 3. 타이포그래피

- **UI 서체**: Pretendard (한글/영문 모두 대응, 웹폰트 무료 라이선스, 사내 도구에 흔히 쓰이는 가독성 우선 서체). 별도 디스플레이 서체는 사용하지 않는다 — 이 서비스는 브랜드 마케팅이 아니라 업무 도구이므로 서체 개성보다 가독성을 우선한다.
- **데이터/시간 표기**: 회의실 시간대 그리드, 캘린더의 숫자(시간, 날짜)는 `tabular-nums`(고정폭 숫자) 스타일을 적용해 자릿수가 바뀌어도 정렬이 흔들리지 않게 한다.

### 타입 스케일

| 용도 | 크기 | 굵기 |
|---|---|---|
| 페이지 타이틀 | 24px | 700 |
| 섹션 타이틀 (모달 헤더 등) | 18px | 600 |
| 본문 | 14px | 400 |
| 캡션/보조 텍스트 | 12px | 400 |
| 버튼 텍스트 | 14px | 600 |

## 4. 레이아웃 패턴

화면/기능 설계서에서 반복적으로 나타나는 UI 패턴 3가지를 공통 컴포넌트로 정의한다.

### 우측 배너 (Side Panel)
캘린더의 날짜별 일정 조회(1-4), 회의실 예약 입력(1-6)에서 공통으로 쓰인다.
- 화면 우측에서 슬라이드로 나타남, 너비 고정(예: 360~400px)
- 배경 딤(dim) 처리는 하지 않는다 — 배너를 연 상태에서도 뒤 화면(캘린더/회의실 목록)을 계속 참고할 수 있어야 하므로.
- 닫기는 우상단 X 버튼 + 배너 바깥 영역 클릭 둘 다 지원.

### 모달 (Modal)
일정 상세 조회(1-4), 일정 추가(1-4)에서 쓰인다.
- 배경 딤 처리 적용(모달은 배너와 달리 다른 작업을 막고 집중을 요구하는 액션이므로 구분).
- 중앙 정렬, 최대 너비 480~560px.
- 닫기는 우상단 X 버튼 + 배너 바깥 영역 클릭 둘 다 지원.

### 시간대 그리드 (Time Grid)
회의실 현황판(1-6)의 9:00~18:00 가로축 예약 표시에 전용으로 쓰인다.
- 1시간 단위 세로 구분선, 예약 블록은 `color_label`이 아닌 회의실 전용 고정 색(회의실 도메인은 일정 색상 라벨과 별개로 취급, 예약 상태별로만 색 구분: 예약완료=`--color-primary`, 취소됨=`--color-text-muted`)
- 가로 스크롤 대신 회의실이 많아지면 세로로 회의실 목록이 늘어나는 구조를 기본으로 한다(시간대 축은 고정 유지).

## 5. 상태 배지 (Status Badge)

`users.status`(마이페이지/조직도)와 `rooms_reservations.status`(회의실)에 공통으로 쓰이는 배지 컴포넌트.

| 상태 | 배지 색 |
|---|---|
| 업무중 (`WORKING`) | `--color-success` 계열 |
| 휴가중 (`ON_LEAVE`) | `--color-text-muted` 계열 (회색, 비활동을 나타냄) |
| 외근/출장중 (`BUSINESS_TRIP`) | `--color-warning` 계열 |
| 예약완료 | `--color-success` 계열 |
| 승인대기 | `--color-warning` 계열 |
| 취소됨 | `--color-text-muted` 계열 |

색상뿐 아니라 텍스트 라벨을 항상 함께 표시한다(색맹 사용자 접근성 — 색으로만 상태를 구분하지 않는다).

## 6. 라이팅 톤 (UI 문구)

- 버튼은 사용자가 통제하는 동작을 그대로 명명한다: "저장" (X), "변경사항 저장" (O) — 무엇이 저장되는지 명확히.
- 성공/실패 메시지는 시스템의 목소리로, 감정 표현 없이 무엇이 어떻게 됐는지만 말한다. 예: "저장되었습니다" (O), "앗! 문제가 발생했어요 😢" (X).
- 에러 메시지는 원인과 다음 행동을 함께 제시한다. 예: "이미 예약된 시간대입니다. 다른 시간을 선택해주세요." (O), "오류가 발생했습니다." (X, 무엇을 해야 할지 알 수 없음).
- 빈 상태(예: 등록된 일정이 없는 날)는 다음 행동을 안내한다. 예: "등록된 일정이 없습니다. 우측 상단 버튼으로 새 일정을 추가하세요."

## 7. 접근성/반응형 최소 기준 (NFR-006 연결)

- NFR-006(PC 및 모바일 환경에서 사용 가능)에 따라, 모든 화면은 최소 375px 너비(모바일 기준)부터 대응한다. 회의실 시간대 그리드처럼 가로 정보가 많은 화면은 모바일에서 가로 스크롤을 허용하되, 핵심 액션(예약 버튼 등)은 스크롤 없이 접근 가능해야 한다.
- 모든 클릭 가능 요소는 키보드 포커스 시 시각적으로 구분되어야 한다(포커스 링 제거 금지).
- 색상 대비: 본문 텍스트와 배경 간 명도 대비 최소 4.5:1을 목표로 한다(특히 보라색 테마의 `--color-text-muted` 값은 실제 구현 시 대비 확인 필요, 위 2번의 `*` 표시 값들과 함께 재검토 대상).
