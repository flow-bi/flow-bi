# 대형 문서 전체 읽기

## 핵심 원칙

- 대상 파일의 경로와 크기를 확인한 후 본문을 읽는다.
- 작업에 필요한 문서만 선정한다.
- 한 번의 도구 호출에서는 한 파일의 본문만 읽는다.
- 여러 대형 문서의 본문을 하나의 출력으로 합치거나 병렬로 읽지 않는다.
- 문서는 첫 줄부터 마지막 줄까지 겹치지 않는 구간으로 읽는다.
- 읽은 파일과 구간을 내부적으로 기록하고, 변경되지 않은 구간은 다시 읽지 않는다.
- 문서 전체를 읽기 전에는 전체 문서에 대한 결론을 내리거나 수정하지 않는다.
- Context에 한번 읽었던 문서가 있다면 다시 읽지 않고, 변경된 구간만 확인한다.
- 작업 범위가 확정되기 전에는 Frontend와 Backend 상세 문서를 모두 읽지 않는다.
- 생성물과 의존성 디렉터리는 탐색 대상에서 제외한다.

## 실행 절차

1. 대상이 명확하지 않으면 `rg --files`로 후보 경로만 확인한다.
2. `wc -l -c`로 대상 문서의 줄 수와 크기를 확인한다.
3. 작업 목적에 필요한 문서만 선정한다.
4. 출력이 잘리지 않을 크기로 읽기 구간을 정한다.
5. 첫 줄부터 마지막 줄까지 구간을 순서대로 읽는다.
6. 각 구간에서 다음 내용을 확인한다.
   - 문서의 목적과 범위
   - 주요 섹션
   - 확정된 결정과 미확정 사항
   - 관련 문서
   - 중복되거나 충돌하는 내용
7. 전체 문서를 읽은 후 수정 대상과 영향 범위를 정리한다.
8. 필요한 수정이나 검토 결과를 작성한다.
9. 수정한 경우 변경 구간과 영향을 받는 관련 섹션의 일관성을 확인한다.

## 구간 읽기 규칙

- 구간은 서로 겹치거나 빠지지 않게 지정한다.
- 문서 앞부분이나 검색 결과만으로 나머지 내용을 추론하지 않는다.
- 출력이 잘리면 전체 파일을 다시 읽지 않는다.
- 출력이 잘린 경우 마지막으로 확인한 줄 다음부터 더 작은 구간으로 이어서 읽는다.
- 이미 읽은 파일은 내용이 변경되지 않은 한 다시 읽지 않는다.
- 문서가 변경된 경우에도 변경 구간과 관련 섹션부터 확인한다.
- 구조 전체가 바뀐 재작성 작업에서만 수정 후 전체 문서를 다시 확인한다.

## 탐색 제외 대상

파일 탐색 시 다음 디렉터리를 기본적으로 제외한다.

- `node_modules`
- `dist`
- `build`
- `.gradle`
- `coverage`
- 기타 자동 생성 디렉터리

## 완료 조건

- 대상 문서의 첫 줄부터 마지막 줄까지 확인했다.
- 모든 주요 섹션을 확인했다.
- 확정 사항과 미확정 사항을 구분했다.
- 관련 문서와 수정 영향 범위를 확인했다.
- 동일한 파일이나 구간을 불필요하게 다시 읽지 않았다.
- 일부 내용만 확인한 상태에서 전체 검토 완료를 주장하지 않았다.

## 실패 조건

다음 중 하나라도 해당하면 완료로 판단하지 않는다.

- 문서의 일부 구간을 읽지 않았다.
- 출력이 잘렸는데 누락된 구간을 이어서 읽지 않았다.
- 검색 결과나 문서 앞부분만으로 전체를 판단했다.
- 여러 대형 문서의 본문을 하나의 출력으로 합쳤다.
- 문서 전체를 읽기 전에 전체 결론을 작성하거나 문서를 수정했다.
- 수정 영향 범위를 확인하지 않았다.
