# AGENTS.md

## 프로젝트 개요

이 프로젝트는 **AI 비서 기능을 포함한 그룹웨어 시스템**이다.

사용자는 자연어로 일정 등록, 알림 설정, 팀 태그, 회의실 예약, 채팅, 조직 조회 등의 업무를 요청할 수 있다.  
시스템은 사용자의 요청을 분석하고, 필요한 경우 확인 단계를 거쳐 실제 업무 데이터를 생성하거나 변경한다.

이 레포지토리는 크게 `frontend`와 `backend`로 나뉜다.

- `frontend`: 화면 구현, 라우팅, 상태 관리, API 연동, UI 컴포넌트 구현
- `backend`: API, 인증/인가, 비즈니스 로직, DB 연동

이 문서는 프로젝트 전체에 적용되는 공통 규칙만 정의한다.

---

## 작업 기본 원칙

- 변경 전 현재 코드 구조와 관련 문서를 먼저 확인한다.
- 기존 설계 방향을 우선 따르고, 새로운 구조를 임의로 만들지 않는다.
- 한 번에 큰 범위를 수정하지 말고, 가능한 작은 단위로 변경한다.
- 요구사항에 없는 기능을 임의로 추가하지 않는다.
- 불확실한 부분은 추측해서 확정하지 말고, 코드나 문서를 기준으로 판단한다.
- 작업 후 변경 내용과 검증 결과를 명확히 남긴다.
- 문서와 코드가 충돌하면 임의로 판단하지 않고 작업 기록에 남긴다.

## 영역별 책임

### Frontend

프론트엔드는 다음을 담당한다.

- 화면 구성
- 라우팅
- 사용자 입력 처리
- API 요청 및 응답 표시
- 로딩, 에러, 빈 상태 처리
- 사용자 권한에 따른 메뉴 노출
- mock 데이터 기반 화면 구현

프론트엔드 작업 중 `backend/`를 임의로 수정하지 않는다.

### Backend

백엔드는 다음을 담당한다.

- API 구현
- 인증/인가
- 비즈니스 로직
- DB 연동
- 데이터 검증
- 실제 권한 검증

백엔드 작업 중 `frontend/`를 임의로 수정하지 않는다.

## 하면 안 되는 것

### 프로젝트 구조 관련

- AGENTS.md를 임의로 수정하지 않는다.
- 최상위 폴더 구조를 임의로 변경하지 않는다.
- 프론트엔드와 백엔드의 책임을 섞지 않는다.
- 도메인 경계를 무시하고 서로 다른 기능의 코드를 한곳에 몰아넣지 않는다.
- 기존 문서와 다른 방향의 구조를 새로 만들지 않는다.

### 코드 변경 관련

- 요구사항과 관련 없는 리팩터링을 하지 않는다.
- 사용하지 않는 라이브러리를 임의로 추가하지 않는다.
- 기존 코딩 스타일과 다른 패턴을 갑자기 도입하지 않는다.
- 타입 오류나 린트 오류를 무시하지 않는다.
- 임시 코드를 남긴 채 작업을 완료하지 않는다.
- `TODO`, `FIXME`를 불필요하게 남기지 않는다.

### 데이터/API 관련

- API 요청/응답 형태를 추측해서 구현하지 않는다.
- 명세되어 있지 않은 필드를 임의로 만들지 않는다.
- 실제 데이터를 변경하는 기능을 사용자 확인 없이 실행하지 않는다.
- 인증, 권한, 조직 정보와 관련된 로직을 단순 UI 조건으로만 처리하지 않는다.
- 민감한 정보나 토큰을 로그에 출력하지 않는다.

### AI 비서 기능 관련

- 자연어 요청을 무조건 하나의 기능으로 단정하지 않는다.
- 일정 생성, 알림 발송, 회의실 예약, 팀 태그, 권한 변경 같은 작업을 확인 없이 바로 실행하지 않는다.
- 사용자의 의도가 애매한 경우 임의로 값을 채워 실행하지 않는다.
- 실패 가능성이 있는 작업을 성공한 것처럼 표시하지 않는다.
- AI 응답만 믿고 실제 시스템 상태 확인을 생략하지 않는다.

### 문서 관련

- 문서를 수정하지 않고 구조나 규칙만 변경하지 않는다.
- 오래된 문서를 그대로 두고 반대되는 새 규칙을 추가하지 않는다.
- 문서와 코드가 충돌하면 작업 기록에 남긴다.
