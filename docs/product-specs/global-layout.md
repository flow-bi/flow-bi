# 전역 레이아웃 Product Spec

## 기능 목적

서비스 전반에서 공통으로 사용하는 헤더와 대표 색상 테마의 적용 기준을 정의한다.

## 상세 요구사항

### 포함한 요구사항

- 헤더 좌측 상단에 기업명을 표시한다.
- 헤더 우측 상단에 로그인한 사용자의 이름을 표시한다.
- 관리자 권한을 가진 사용자는 대표 색상 테마를 선택할 수 있다.
- 선택한 대표 색상 테마는 회사 단위의 전역 설정으로 적용한다.
- 후보 테마는 흰색/하늘색(`#EDF1F6`), 보라색/흰색/연보라(`#5541A4`)이다.

### 제외한 요구사항

- 각 화면의 본문 구성은 해당 기능별 Product Spec에 둔다.
- 공통 헤더에는 검색창을 제공하지 않는다.
- 전역 검색 기능은 검색 대상과 권한 정책을 별도 기능으로 정의한 후 추가한다.
- 일반 사용자의 개인별 테마 설정은 제공하지 않는다.

## 화면 구성 요소

- 공통 헤더
  - 기업명
  - 본인 이름
- 관리자용 테마 선택 요소
  - 흰색/하늘색 테마
  - 보라색/흰색/연보라 테마

## 데이터 흐름

1. 인증된 사용자의 기업명과 이름을 헤더에 표시한다.
2. 관리자 권한을 가진 사용자가 대표 색상 테마를 선택한다.
3. 선택한 테마를 회사 단위의 전역 설정으로 저장한다.
4. 저장된 테마를 서비스의 모든 주요 화면에 동일하게 적용한다.
5. 일반 사용자는 회사에 설정된 대표 색상 테마를 적용받으며 개별적으로 변경할 수 없다.

## 확정된 기능 결정

- 헤더는 모든 주요 화면에 공통으로 존재하는 전역 요소로 본다.
- 기업명과 본인 이름은 헤더의 필수 표시 요소로 본다.
- 공통 헤더에는 검색창을 제공하지 않는다.
- 전역 검색이 필요한 경우 검색 대상과 접근 권한 정책을 별도 기능으로 정의한 후 추가한다.
- 대표 색상 테마는 회사 단위의 전역 UI 설정으로 관리한다.
- 대표 색상 테마는 관리자 권한을 가진 사용자만 선택하고 변경할 수 있다.
- 일반 사용자는 대표 색상 테마를 개별적으로 변경할 수 없다.
- 대표 색상 테마 후보는 흰색/하늘색과 보라색/흰색/연보라 조합으로 구성한다.

## 확정된 기능 결정의 근거

- `ui-ux-spec.md` 1-0에서 헤더의 좌측 기업명과 우측 본인 이름을 명시한다.
- 검색창의 용도와 검색 대상이 정의되어 있지 않으므로 기능이 없는 입력 요소를 제공하지 않는다.
- 전역 검색은 검색 대상별 접근 권한과 결과 표시 정책이 필요하므로 별도 기능으로 정의한다.
- `ui-ux-spec.md` 1-9에서 대표 색상과 선택 가능한 색상 조합을 명시한다.
- 대표 색상을 회사 단위로 동일하게 적용하여 서비스 전반의 UI 일관성을 유지한다.
- 일반 사용자의 개별 테마 설정을 제외하여 회사의 대표 색상 기준을 일관되게 적용한다.

## Scope Decision

- In Scope: 기업명 및 사용자 이름 표시, 관리자용 대표 색상 테마 선택, 회사 단위 전역 테마 적용.
- Out of Scope: 전역 검색, 개별 화면 레이아웃, 일반 사용자의 개인별 테마 설정.
- Moved: 사용자 이름과 기업 정보의 원천 데이터는 `auth.md`, `my-page.md`, `organization-chart.md`에서 관리한다. 전역 검색은 검색 대상과 권한 정책을 정의하는 별도 기능으로 이동한다.
