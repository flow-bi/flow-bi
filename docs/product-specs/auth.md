# 인증 Product Spec

## 기능 목적

사용자가 사번과 비밀번호로 그룹웨어에 로그인하고, 인증된 사용자만 서비스를 이용할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 사용자는 아이디와 비밀번호를 이용하여 로그인할 수 있어야 한다. (FR-009)
- 사용자는 로그아웃할 수 있어야 한다. (FR-010)
- 로그인 화면은 아이디(사번), 비밀번호, 비밀번호 변경, 로그인 요소를 제공한다.
- 로그인 사용자만 서비스를 이용 가능해야 한다. (NFR-001)
- 비밀번호는 암호화하여 저장해야 한다. (NFR-002)

### 제외한 요구사항

- 직원 등록, 조회, 수정, 삭제는 `admin.md`로 이동한다.
- 개인정보 조회 및 수정은 `my-page.md`로 이동한다.

## 화면 구성 요소

- 로그인 화면
  - 아이디(사번) 입력
  - 비밀번호 입력
  - 비밀번호 변경 진입 요소
  - 로그인 실행 요소
- 인증 후 공통 화면
  - 로그아웃 실행 요소

## 데이터 흐름

1. 사용자가 사번과 비밀번호를 입력한다.
2. 입력된 사번은 임직원 정보와 연결된 인증 정보를 식별하는 데 사용된다.
3. 비밀번호 검증 후 로그인 상태가 생성된다.
4. 인증된 사용자는 서비스 화면에 접근한다.
5. 로그아웃 시 사용자의 로그인 상태가 종료된다.

## 확정된 기능 결정

- 사번은 로그인 아이디로 사용한다.
- 사용자는 사번과 비밀번호로 로그인할 수 있다.
- 사용자는 로그아웃할 수 있다.
- 로그인한 사용자만 서비스를 이용할 수 있다.
- 로그인 화면에는 비밀번호 변경 진입 요소가 있다.

## 확정된 기능 결정의 근거

- `ui-ux-spec.md` 1-1에서 아이디가 사번이라고 명시되어 있다.
- `ui-ux-spec.md` 1-1에서 비밀번호 변경과 로그인을 로그인 화면 요소로 정의한다.
- `requirements.md` FR-009, FR-010에서 로그인과 로그아웃을 요구한다.
- `requirements.md` NFR-001에서 로그인 사용자만 서비스 이용 가능해야 한다고 정의한다.

## Scope Decision

- In Scope: 로그인, 로그아웃, 비밀번호 변경 진입, 인증 사용자 접근 제한, 비밀번호 암호화 저장.
- Out of Scope: 직원 계정 등록/수정/삭제, 프로필 정보 수정, 권한별 관리자 메뉴 구성.
- Moved: 직원 계정 관리는 관리자 기능에 속하므로 `admin.md`로 이동한다.

## 미확정 사항

- 비밀번호 변경의 상세 절차와 화면 범위는 원문에 정의되어 있지 않다.
- 자체 로그인 외 인증 제공자 사용 여부는 `database.md`에서 질문으로 남아 있다.
