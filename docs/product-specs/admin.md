# 관리 Product Spec

## 기능 목적

관리자가 직원과 팀 정보를 등록, 조회, 수정, 삭제 또는 비활성화하여 조직 운영 데이터를 관리할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 관리자는 직원 정보를 등록할 수 있어야 한다. (FR-001)
- 관리자는 등록된 직원 목록을 조회할 수 있어야 한다. (FR-002)
- 관리자는 직원 정보를 수정할 수 있어야 한다. (FR-003)
- 관리자는 직원을 비활성화 또는 삭제할 수 있어야 한다. (FR-004)
- 관리자는 팀을 생성할 수 있어야 한다. (FR-005)
- 관리자는 팀 목록을 조회할 수 있어야 한다. (FR-006)
- 관리자는 팀 정보를 수정할 수 있어야 한다. (FR-007)
- 관리자는 팀을 삭제할 수 있어야 한다. (FR-008)
- 관리자용 탭이 별도로 존재한다.
- 관리자 탭에는 직원관리와 팀관리가 있다.
- 직원관리에서는 직원 목록을 표시하고 수정/삭제가 가능하다.
- 팀관리에서는 팀 목록을 표시하고 수정/삭제가 가능하다.

### 제외한 요구사항

- 일반 사용자의 본인 정보 수정은 `my-page.md`로 이동한다.
- 조직도에서 직원 정보를 조회하는 기능은 `organization-chart.md`로 이동한다.
- 로그인 자체는 `auth.md`로 이동한다.

## 화면 구성 요소

- 관리자용 탭
- 직원관리 탭
  - 직원 목록
  - 직원 등록
  - 직원 수정
  - 직원 삭제 또는 비활성화
- 팀관리 탭
  - 팀 목록
  - 팀 생성
  - 팀 수정
  - 팀 삭제

## 데이터 흐름

1. 관리자 권한을 가진 사용자가 관리 탭에 접근한다.
2. 직원관리에서 직원 목록을 조회하고 직원 정보를 등록, 수정, 삭제 또는 비활성화한다.
3. 팀관리에서 팀 목록을 조회하고 팀 정보를 생성, 수정, 삭제한다.
4. 직원 정보는 팀 및 직급 정보와 연결된다.
5. 팀 정보 변경은 조직 계층 정보에 반영된다.

## 확정된 기능 결정

- 관리 기능은 관리자용 탭으로 분리한다.
- 직원과 팀 관리는 별도 탭으로 구분한다.
- 직원은 팀과 직급을 가진다.
- 팀은 계층 구조를 가질 수 있다.
- 관리자용 탭은 관리자 권한을 가진 사용자가 접근하는 기능으로 본다.

## 확정된 기능 결정의 근거

- `requirements.md` FR-001~008에서 직원 및 팀 관리 요구사항을 정의한다.
- `ui-ux-spec.md` 1-7에서 관리자용 탭, 직원관리, 팀관리를 정의한다.
- `database.md`의 `users`, `teams`, `teams_closure`, `positions`는 직원과 조직 관리 데이터를 정의한다.
- `ui-ux-spec.md` 1-7에서 관리 기능이 관리자용 탭으로 별도 존재한다고 정의한다.

## Scope Decision

- In Scope: 직원 등록/조회/수정/삭제 또는 비활성화, 팀 생성/조회/수정/삭제, 관리자 탭.
- Out of Scope: 본인 정보 수정, 조직도 탐색, 인증 절차.
- Moved: 본인 정보 관리는 `my-page.md`, 조직 구조 조회는 `organization-chart.md`, 로그인은 `auth.md`로 이동한다.

## 미확정 사항

- 직원 삭제와 비활성화 중 어떤 동작을 기본 정책으로 할지 확정되어 있지 않다.
- 관리자 권한의 세부 역할과 권한 목록은 정의되어 있지 않다.
- 팀 삭제 시 하위 팀과 소속 직원 처리 정책은 정의되어 있지 않다.
