# header.md

## 개요
모든 화면 상단에 공통으로 표시되는 헤더. 특정 백엔드 도메인에 속하지 않는 공통 UI 요소이며, `user` 도메인(본인 정보)에 일부 의존한다.

## 구성 요소

| 위치 | 요소 | 데이터 출처 | 비고 |
|---|---|---|---|
| 좌측 상단 (모바일) | 사이드바 토글(햄버거) | 없음 | 모바일에서만 표시, 사이드바 오버레이 열기 |
| 좌측 상단 | 기업명 | 정적 텍스트 | 백엔드 연동 없음 |
| 우측 상단 | 본인 이름 | `user` 도메인 (로그인 사용자 정보) | 로그인 세션에서 가져옴 |
| 우측 상단 | 검색창 | 없음 (UI만) | 기능 비활성화, 아래 참고 |

## 모바일 사이드바 토글 (확정)

`layout.md`의 모바일 반응형 요구사항에 따라 헤더 좌측에 햄버거 버튼을 추가한다.

- 데스크톱에서는 사이드바가 상시 노출되므로 햄버거 버튼을 숨긴다.
- 모바일에서는 햄버거 버튼을 표시하고, 누르면 사이드바를 오버레이 형태로 연다.
- 아이콘 세트가 아직 정해지지 않았으므로, MVP1에서는 별도 아이콘 라이브러리 없이 CSS 라인 3개로 햄버거 모양을 만든다.

## 미해결 이슈: 검색창 용도 미정

화면/기능 설계서 원문에 "검색창? (용도는?)"으로만 표시되어 있어 다음 중 무엇을 검색하는 화면인지 정의되어 있지 않다:

- 조직도 내 직원 검색?
- 일정 검색?
- 전체 통합 검색(일정+직원+회의실)?
- MVP2의 AI 비서/문서 검색과 연결되는 자리?

## 검색창 처리 방침 (확정)

검색창의 정확한 용도는 아직 정의되지 않았다. **MVP1에서는 UI만 배치하고 기능은 비활성화**한다 (클릭/입력해도 동작 없음). 용도가 확정되면 이 문서를 갱신하고 해당 도메인에 검색 API를 추가한다.
