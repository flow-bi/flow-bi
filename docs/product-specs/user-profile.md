# user-profile.md

**대응 도메인**: `user`
**대응 화면**: 화면/기능 설계서 1-8. 마이페이지
**관련 FR**: FR-024(프로필 사진 변경), FR-025(개인정보 수정)

## 기능 요구사항

사용자는 본인의 프로필 정보를 조회하고, 일부 항목을 수정할 수 있다.

### 조회/수정 가능 항목 (화면설계서 1-8 기준)

| 항목 | 변경 가능 여부(화면설계서) | 비고 |
|---|---|---|
| 프로필 사진 | 변경 불가 (확정) | 기본 이미지만 사용, FR-024 미채택 |
| 이름 | 변경안됨 | |
| 사번 | 변경안됨 | |
| 이메일 | **변경됨** | FR-025 |
| 사내 전화번호 | **변경됨** | FR-025 |
| 직급/직책 | 변경안됨 | |
| 활성 상태 (업무중/휴가중/외근·출장중) | **변경됨** | 화면설계서에 명시, FR 번호 별도 매핑 없음 |

## 프로필 사진 정책 (확정)

프로필 사진은 **변경할 수 없다**. 모든 사용자는 기본 이미지(default image)만 사용하며, 업로드/변경 기능 자체를 제공하지 않는다. `users.profile_image_url`은 시딩 시 기본 이미지 경로로 일괄 설정하고, 이를 변경하는 API는 만들지 않는다. 요구사항정의서 FR-024(프로필 사진 변경)는 이 서비스에서는 채택하지 않는 것으로 확정한다.

## API 개요 (초안)

| Method | Endpoint | 설명 |
|---|---|---|
| GET | `/api/v1/users/me` | 본인 프로필 조회 (이름, 사번, 이메일, 전화번호, 직급, 팀, 활성상태, 프로필사진 URL 등) |
| PATCH | `/api/v1/users/me` | 이메일/전화번호/활성상태 수정 (프로필 사진 제외) |

### Request Body 예시 (PATCH)
```json
{
  "email": "user@company.com",
  "phone_number": "010-1234-5678",
  "status": "휴가중"
}
```
- `email`, `phone_number`, `status`는 모두 선택적(partial update)으로 처리한다.
- `name`, `employee_number`, `position_id`, `team_id`, `profile_image_url`은 이 API로 수정할 수 없다(관리자 시딩 데이터 영역).

## 활성 상태(status) 값 정의 필요

화면설계서에 "업무중 / 휴가중 / 외근·출장중" 세 가지 값이 예시로 제시되어 있다. `users.status` 컬럼(VARCHAR(30))에 저장할 코드값(enum)을 확정해야 한다.

**가정(확인 필요)**: `WORKING`, `ON_LEAVE`, `BUSINESS_TRIP` 3개 값으로 시작. 추가 상태(예: 재택근무)가 필요하면 이 문서와 `db-schema.md`를 갱신한다.
