# 마이페이지 Product Spec

## 기능 목적

사용자가 본인 기본 정보를 확인하고, 허용된 개인정보와 활성 상태 및 알림 설정을 관리할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 사용자는 전화번호, 이메일, 위치 정보를 수정할 수 있어야 한다. (FR-025)
- 사용자는 알림 사용 여부를 설정할 수 있어야 한다. (FR-026)
- 사용자는 일정 시작 전 알림 시간을 설정할 수 있어야 한다. (FR-027)
- 사용자는 알림 종류를 설정할 수 있어야 한다. (FR-028)
- 마이페이지에는 프로필 사진, 이름, 기본정보, 직급/직책, 활성 상태를 표시한다.
- 이름, 사번, 직급/직책은 변경되지 않는다.
- 이메일과 사내 전화번호는 변경된다.
- 활성 상태는 변경된다.
- 활성 상태 후보는 업무중, 휴가중, 외근/출장중이다.

### 제외한 요구사항

- 직원 전체 관리 기능은 `admin.md`로 이동한다.
- 조직도에서 직원 정보를 조회하는 기능은 `organization-chart.md`로 이동한다.
- 알림 발송 자체는 `notification.md`로 이동한다.

## 화면 구성 요소

- 프로필 사진
- 이름
- 기본정보
  - 사번
  - 이메일
  - 사내 전화번호
- 직급/직책
- 활성 상태
  - 업무중
  - 휴가중
  - 외근/출장중
- 알림 설정
  - 알림 사용 여부
  - 일정 시작 전 알림 시간
  - 알림 종류

## 데이터 흐름

1. 사용자가 마이페이지에 진입하면 본인 기본 정보를 조회한다.
2. 사용자가 변경 가능한 개인정보를 수정한다.
3. 사용자가 활성 상태를 변경한다.
4. 사용자가 알림 사용 여부, 알림 시간, 알림 종류를 설정한다.
5. 알림 설정은 일정 및 AI 비서 알림 제공 시 기준으로 사용된다.

## 확정된 기능 결정

- 마이페이지는 본인 정보 관리 화면이다.
- 이름, 사번, 직급/직책은 변경 불가 항목이다.
- 이메일, 사내 전화번호, 활성 상태는 변경 가능 항목이다.
- 알림 설정은 사용자가 직접 조정할 수 있다.

## 확정된 기능 결정의 근거

- `requirements.md` FR-025~028에서 개인정보 수정과 알림 설정 요구사항을 정의한다.
- `ui-ux-spec.md` 1-8에서 마이페이지 표시 항목과 변경 가능 여부를 정의한다.
- `database.md`의 `users`는 이메일, 전화번호, 상태, 프로필 이미지 등 사용자 정보를 정의한다.

## Scope Decision

- In Scope: 본인 정보 조회, 이메일/전화번호 수정, 활성 상태 변경, 알림 설정.
- Out of Scope: 직원 전체 관리, 조직도 조회, 알림 발송 로직.
- Moved: 직원 전체 관리는 `admin.md`, 조직도 조회는 `organization-chart.md`, 알림 제공은 `notification.md`로 이동한다.

## Reference Mapping

- `docs/references/ui-ux-spec.md`: 1-8 마이페이지

## 미확정 사항

- `requirements.md` FR-024는 프로필 사진 변경을 요구하지만, `ui-ux-spec.md` 1-8은 프로필 사진이 변경되지 않는다고 명시한다.
- `requirements.md` FR-025의 위치 정보 수정은 마이페이지 화면 항목에 나타나지 않는다.
- 알림 종류의 구체적인 선택지가 정의되어 있지 않다.
