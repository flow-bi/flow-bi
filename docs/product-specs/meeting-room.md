# 회의실 Product Spec

## 기능 목적

사용자가 회의실 목록과 예약 현황을 확인하고, 원하는 시간에 회의실을 예약, 수정, 취소할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 사용자는 예약 가능한 회의실 목록을 조회할 수 있어야 한다. (FR-019)
- 사용자는 회의실을 예약할 수 있어야 한다. (FR-020)
- 회의실 예약 시 일정이 자동 생성되어 캘린더에 표시되어야 한다. (FR-021)
- 사용자는 회의실 예약 정보를 수정할 수 있어야 한다. (FR-022)
- 사용자는 회의실 예약을 취소할 수 있어야 한다. (FR-023)
- 회의실 목록에는 회의실 이름, 사진, 수용 인원수, 장비를 표시한다.
- 회의실별 예약 현황을 9:00~18:00 시간대 기준으로 가로 표시한다.
- 예약 현황에는 예약 제목, 시간대, 예약 팀을 표시한다.
- 초기에는 전체 회의실을 표시한다.
- 검색 조건은 수용인원, 날짜, 시간대, 상태이다.
- 검색 결과는 해당 내용만 필터링하는 방식이 아니라 우선순위가 변경되어 조회된다.
- 원하는 회의실 선택 후 우측 배너에서 예약한다.
- 예약 시 선택된 회의실 정보, 날짜, 시간대, 참석자, 회의 일정 상세 설명을 입력한다.

### 제외한 요구사항

- 캘린더의 일반 일정 생성/수정/삭제는 `calendar.md`로 이동한다.
- AI를 통한 회의실 예약 지원은 `ai-assistant.md`로 이동한다.
- 회의실 예약 알림은 `notification.md`로 이동한다.

## 화면 구성 요소

- 회의실 목록
  - 회의실 이름
  - 회의실 사진
  - 수용 인원수
  - 장비
- 회의실 예약 현황
  - 회의실별 시간대 표시
  - 예약 제목
  - 예약 시간대
  - 예약 팀
- 회의실 검색 조건
  - 수용인원
  - 날짜
  - 시간대
  - 상태
- 예약 우측 배너
  - 선택된 회의실 정보
  - 날짜
  - 시간대
  - 참석자 검색 및 추가
  - 회의 일정 상세 설명

## 데이터 흐름

1. 회의실 화면 진입 시 전체 회의실 목록과 예약 현황을 조회한다.
2. 사용자가 검색 조건을 입력하면 회의실 목록의 표시 우선순위가 조정된다.
3. 사용자가 회의실을 선택하면 예약 입력 배너가 표시된다.
4. 예약 정보 제출 시 회의실 예약 정보가 생성된다.
5. 회의실 예약과 연결된 일정이 자동 생성되어 캘린더에 표시된다.
6. 예약 수정 시 연결된 일정과 예약 시간이 함께 일관성을 유지한다.
7. 예약 취소 시 예약 상태와 취소 시점이 관리된다.
8. 예약 생성/수정/취소 시 일정 및 알림 연동 정책은 `docs/design-docs/schedule-and-notification.md`를 따른다.

## 확정된 기능 결정

- 회의실 예약은 일정과 연결된다.
- 동일 회의실의 겹치는 시간대 예약은 허용되지 않는다.
- 회의실 검색은 완전 필터링이 아니라 우선순위 변경 방식이다.
- 회의실 예약 현황은 9:00~18:00 시간대 기준으로 표시한다.
- 예약 생성/수정/취소 시 일정 및 알림 연동 정책은 `docs/design-docs/schedule-and-notification.md`를 따른다.

## 확정된 기능 결정의 근거

- `requirements.md` FR-019~023에서 회의실 목록, 예약, 일정 자동 생성, 수정, 취소를 정의한다.
- `ui-ux-spec.md` 1-6에서 회의실 목록, 현황, 검색, 예약 입력 항목을 정의한다.
- `database.md`의 `rooms`와 `rooms_reservations`는 회의실과 예약 데이터를 정의한다.
- `database.md`는 예약이 `schedules`와 연결되어 캘린더 일정과 동기화된다고 설명한다.
- 예약과 일정, 알림의 상세 연동 정책은 `docs/design-docs/schedule-and-notification.md`에서 다룬다.

## Scope Decision

- In Scope: 회의실 목록, 현황, 검색, 예약, 수정, 취소, 일정 자동 생성.
- Out of Scope: 일반 일정 CRUD, AI 예약 대화, 알림 발송.
- Moved: 캘린더 화면 표시는 `calendar.md`, AI 예약 지원은 `ai-assistant.md`, 예약 알림은 `notification.md`로 이동한다. 예약과 일정, 알림의 연동 정책은 `docs/design-docs/schedule-and-notification.md`로 이동한다.

## 미확정 사항

- 회의실 사진의 필수 여부와 이미지 출처가 정의되어 있지 않다.
- 장비 정보의 구조가 텍스트라고만 정의되어 있고 세부 형식은 정해지지 않았다.
- 예약 상태 값의 정확한 종류는 정의되어 있지 않다.
- 예약 수정/취소 시 연결된 일정과 알림을 처리하는 세부 정책은 `docs/design-docs/schedule-and-notification.md`의 미확정 사항으로 관리한다.
