# 회의실 Product Spec

## 기능 목적

사용자가 회의실 목록과 예약 현황을 확인하고, 원하는 시간에 회의실을 예약, 수정, 취소할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 사용자는 예약 가능한 회의실 목록을 조회할 수 있어야 한다. (FR-019)
- 사용자는 회의실을 예약할 수 있어야 한다. (FR-020)
- 회의실 예약 시 일정이 자동 생성되어 캘린더에 표시되어야 한다. (FR-021)
- 사용자는 회의실 예약 정보를 수정할 수 있어야 한다. (FR-022)
- 사용자는 회의실 예약을 취소할 수 있어야 한다. (FR-023)
- 회의실 목록에는 회의실 이름, 사진, 수용 인원수, 장비를 표시한다.
- 회의실 사진은 선택 입력으로 관리한다.
- 등록된 회의실 사진이 없는 경우 기본 회의실 이미지를 표시한다.
- 회의실 사진은 관리자가 업로드한 이미지를 사용한다.
- 회의실 장비는 사전에 정의된 장비 목록에서 복수 선택하여 관리한다.
- 관리자는 장비 종류를 추가하거나 수정할 수 있다.
- 회의실별 예약 현황을 9:00~18:00 시간대 기준으로 가로 표시한다.
- 예약 현황에는 예약 제목, 시간대, 예약 팀을 표시한다.
- 초기에는 전체 회의실을 표시한다.
- 검색 조건은 수용인원, 날짜, 시간대, 상태이다.
- 검색 결과는 해당 내용만 필터링하는 방식이 아니라 우선순위가 변경되어 조회된다.
- 원하는 회의실 선택 후 우측 배너에서 예약한다.
- 예약 시 선택된 회의실 정보, 날짜, 시간대, 참석자, 회의 일정 상세 설명을 입력한다.
- 예약 데이터의 저장 상태는 `RESERVED`, `CANCELED`로 관리한다.
- 화면에 표시되는 예약 예정, 사용 중 상태는 예약 시작·종료 시간과 현재 시간을 기준으로 계산한다.

### 제외한 요구사항

- 캘린더의 일반 일정 생성/수정/삭제는 `calendar.md`로 이동한다.
- AI를 통한 회의실 예약 지원은 `ai-assistant.md`로 이동한다.
- 회의실 예약 알림은 `notification.md`로 이동한다.

## 화면 구성 요소

- 회의실 목록
  - 회의실 이름
  - 회의실 사진
  - 수용 인원수
  - 장비
- 회의실 예약 현황
  - 회의실별 시간대 표시
  - 예약 제목
  - 예약 시간대
  - 예약 팀
  - 예약 상태
- 회의실 검색 조건
  - 수용인원
  - 날짜
  - 시간대
  - 상태
- 예약 우측 배너
  - 선택된 회의실 정보
  - 날짜
  - 시간대
  - 참석자 검색 및 추가
  - 회의 일정 상세 설명
- 관리자용 회의실 관리 요소
  - 회의실 사진 업로드
  - 장비 종류 추가 및 수정
  - 회의실별 장비 복수 선택

## 데이터 흐름

1. 회의실 화면 진입 시 전체 회의실 목록과 예약 현황을 조회한다.
2. 회의실 사진이 등록되어 있으면 관리자가 업로드한 이미지를 표시한다.
3. 회의실 사진이 등록되어 있지 않으면 기본 회의실 이미지를 표시한다.
4. 회의실별로 등록된 장비 목록을 조회하여 표시한다.
5. 사용자가 검색 조건을 입력하면 회의실 목록의 표시 우선순위가 조정된다.
6. 사용자가 회의실을 선택하면 예약 입력 배너가 표시된다.
7. 예약 정보 제출 시 회의실 예약 정보가 `RESERVED` 상태로 생성된다.
8. 회의실 예약과 연결된 일정이 자동 생성되어 캘린더에 표시된다.
9. 예약 수정 시 연결된 일정의 회의실 관련 정보도 함께 수정된다.
10. 예약 취소 시 예약 상태를 `CANCELED`로 변경하고 취소 시점을 관리한다.
11. 화면에 표시되는 예약 예정과 사용 중 상태는 예약 시작·종료 시간과 현재 시간을 기준으로 계산한다.
12. 예약 생성/수정/취소 시 일정 및 알림 연동 정책은 `docs/design-docs/schedule-and-notification.md`를 따른다.

## 확정된 기능 결정

- 회의실 예약은 일정과 연결된다.
- 동일 회의실의 겹치는 시간대 예약은 허용되지 않는다.
- 회의실 검색은 완전 필터링이 아니라 우선순위 변경 방식이다.
- 회의실 예약 현황은 9:00~18:00 시간대 기준으로 표시한다.
- 회의실 사진은 선택 입력으로 관리한다.
- 등록된 회의실 사진이 없는 경우 기본 회의실 이미지를 표시한다.
- 회의실 사진은 관리자가 업로드한 이미지를 사용한다.
- 회의실 장비는 사전에 정의된 장비 목록에서 복수 선택하여 관리한다.
- 관리자는 필요한 경우 장비 종류를 추가하거나 수정할 수 있다.
- 예약 데이터의 저장 상태는 `RESERVED`, `CANCELED`로 관리한다.
- `RESERVED`는 정상적으로 유지 중인 예약을 의미한다.
- `CANCELED`는 취소된 예약을 의미한다.
- 화면에 표시되는 예약 예정과 사용 중 상태는 별도로 저장하지 않는다.
- 예약 예정과 사용 중 상태는 예약 시작·종료 시간과 현재 시간을 기준으로 계산한다.
- 예약 생성/수정/취소 시 일정 및 알림 연동 정책은 `docs/design-docs/schedule-and-notification.md`를 따른다.

## 확정된 기능 결정의 근거

- `requirements.md` FR-019~023에서 회의실 목록, 예약, 일정 자동 생성, 수정, 취소를 정의한다.
- `ui-ux-spec.md` 1-6에서 회의실 목록, 현황, 검색, 예약 입력 항목을 정의한다.
- `database.md`의 `rooms`와 `rooms_reservations`는 회의실과 예약 데이터를 정의한다.
- `database.md`는 예약이 `schedules`와 연결되어 캘린더 일정과 동기화된다고 설명한다.
- 회의실 사진을 선택 입력으로 관리하여 사진이 준비되지 않은 회의실도 등록할 수 있도록 한다.
- 사진이 없는 경우 기본 이미지를 표시하여 회의실 목록의 화면 구조를 일관되게 유지한다.
- 장비 정보를 사전에 정의된 목록으로 관리하여 회의실별 장비 정보를 일관된 형식으로 표시하고 검색할 수 있도록 한다.
- 예약의 저장 상태를 정상 예약과 취소 상태로 단순화하고 시간에 따라 달라지는 화면 상태는 현재 시간을 기준으로 계산한다.
- 예약과 일정, 알림의 상세 연동 정책은 `docs/design-docs/schedule-and-notification.md`에서 다룬다.

## Scope Decision

- In Scope: 회의실 목록, 사진 표시, 장비 표시, 예약 현황, 검색, 예약, 수정, 취소, 일정 자동 생성, 예약 상태 관리.
- Out of Scope: 일반 일정 CRUD, AI 예약 대화, 알림 발송.
- Moved: 캘린더 화면 표시는 `calendar.md`, AI 예약 지원은 `ai-assistant.md`, 예약 알림은 `notification.md`로 이동한다. 예약과 일정, 알림의 연동 정책은 `docs/design-docs/schedule-and-notification.md`로 이동한다.
