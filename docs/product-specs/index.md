## 설계 확정 사항 (기존 미해결 이슈 해소)

1. **프로필 사진**: 변경 불가로 확정. 기본 이미지만 사용하며 FR-024는 채택하지 않는다. → `user-profile.md`
2. **헤더 검색창**: 용도 미정 상태로, MVP1에서는 UI만 배치하고 기능은 비활성화한다. → `header.md`
3. **캘린더 "공개 범위" UI**: `visibility` 컬럼은 ERD 원래 의도(공개/비공개/팀내공개)로 유지하고, 화면의 "공개 범위(개인/팀/프로젝트)"는 공유 대상 지정(`schedule_targets`)으로 재해석한다. `schedules.color_label` 컬럼을 신규 추가한다. → `schedule.md`, `db-schema.md`
4. **회의실 예약 취소**: 연동된 `schedules`도 함께 삭제한다. → `meeting-room.md`
5. **대시보드**: AI 요약만 디자인만이며, 주간 일정/오늘의 일정/팀원 상태는 기존 API를 재사용한 실기능으로 구현한다. → `dashboard.md`

## 남아있는 미해결 이슈

- **알림 설정(FR-026~028)의 화면 존재 여부**: 요구사항정의서에는 있으나 화면/기능 설계서(MVP)의 9개 화면 목록에는 알림 설정 화면이 없다. MVP1 스코프에서 완전히 제외된 것인지, 화면 설계서 작성 시 누락된 것인지 확인이 필요하다.
- **"회의실 일정" 등록 흐름**: 캘린더의 "일정 추가" 모달과 회의실 화면의 "예약하기" 플로우가 같은 결과를 만드는 것인지, 진입점을 하나로 통일할지 확인 필요. → `schedule.md` 3번 참고
