# 일정 관리 Product Spec

## 기능 목적

사용자가 개인, 팀, 프로젝트 일정을 월간/주간/일간 캘린더에서 조회하고 등록, 수정, 삭제할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 사용자는 월 단위로 일정을 조회할 수 있어야 한다. (FR-011)
- 사용자는 주 단위로 일정을 조회할 수 있어야 한다. (FR-012)
- 사용자는 일 단위로 일정을 조회할 수 있어야 한다. (FR-013)
- 사용자는 일정을 등록할 수 있어야 한다. (FR-014)
- 일정 제목, 위치, 날짜, 시간, 참석 인원 정보를 입력할 수 있어야 한다. (FR-015)
- 개인 일정, 팀 일정, 프로젝트 일정 중 하나를 선택할 수 있어야 한다. (FR-016)
- 사용자는 등록한 일정을 수정할 수 있어야 한다. (FR-017)
- 사용자는 등록한 일정을 삭제할 수 있어야 한다. (FR-018)
- 첫 화면은 월간 캘린더이다.
- 월간/주간/일간 캘린더를 선택할 수 있다.
- 날짜 클릭 시 해당 날짜의 일간 일정이 우측 배너에 표시된다.
- 특정 일정 클릭 시 일정 상세가 모달로 표시된다.
- 일정 추가는 모달로 제공한다.
- 일정 추가 항목은 일정 타입, 제목, 날짜, 시작 시간, 종료 시간, 하루종일, 위치, 일정 유형, 공개 범위, 색상 라벨, 참석자, 상세 설명이다.

### 제외한 요구사항

- 회의실 예약 자체는 `meeting-room.md`로 이동한다.
- AI를 통한 자연어 일정 생성은 `ai-assistant.md`로 이동한다.
- 일정 알림 정책 설정은 `my-page.md`로 이동한다.

## 화면 구성 요소

- 월간 캘린더
- 주간 캘린더
- 일간 캘린더
- 캘린더 보기 전환 요소
- 날짜별 일간 일정 우측 배너
- 일정 상세 모달
- 일정 추가 모달
  - 일정 타입
  - 제목
  - 날짜
  - 시작 시간
  - 종료 시간
  - 하루종일
  - 위치
  - 일정 유형
  - 공개 범위
  - 색상 라벨
  - 참석자 검색 및 추가
  - 일정 상세 설명

## 데이터 흐름

1. 사용자가 캘린더 보기 단위를 선택한다.
2. 선택한 월, 주, 일 기준으로 사용자가 볼 수 있는 일정을 조회한다.
3. 날짜 선택 시 해당 날짜의 일정을 우측 배너에 표시한다.
4. 일정 선택 시 상세 정보를 모달에 표시한다.
5. 일정 등록 시 기본 일정 정보, 상세 정보, 공유 대상 정보가 함께 생성된다.
6. 일정 수정 또는 삭제 시 등록자 기준으로 변경 가능한 일정을 대상으로 처리한다.
7. 회의실 예약으로 생성된 일정은 캘린더에 표시된다.
8. 회의실 예약과 일정의 상세 연동 정책은 `docs/design-docs/schedule-and-notification.md`를 따른다.

## 확정된 기능 결정

- 일정은 개인, 팀, 프로젝트 유형을 가진다.
- 일정은 제목, 시작/종료 일시, 등록자를 기본 정보로 가진다.
- 일정 상세 설명과 위치는 상세 정보로 관리한다.
- 일정 공유 대상은 사용자, 팀, 프로젝트 단위로 지정될 수 있다.
- 회의실 예약으로 생성된 일정은 캘린더에서 조회 가능해야 한다.

## 확정된 기능 결정의 근거

- `requirements.md` FR-011~018에서 캘린더 조회 및 일정 CRUD를 정의한다.
- `ui-ux-spec.md` 1-4에서 캘린더 화면, 배너, 모달, 입력 항목을 정의한다.
- `database.md`의 `schedules`, `schedules_details`, `schedule_targets`는 일정 기본 정보, 상세 정보, 공유 대상을 분리해 설명한다.
- `database.md`는 회의실 예약이 일정과 연결되어 캘린더에 자동 추가된다고 설명한다.
- 회의실 예약과 일정의 생성, 수정, 취소 연동 정책은 `docs/design-docs/schedule-and-notification.md`에서 다룬다.

## Scope Decision

- In Scope: 캘린더 조회, 일정 상세 조회, 일정 등록/수정/삭제, 일정 유형 및 공개 범위, 참석자 지정.
- Out of Scope: 회의실 목록/예약 현황, AI 자연어 처리, 알림 발송.
- Moved: 회의실 예약은 `meeting-room.md`, AI 일정 생성은 `ai-assistant.md`, 알림은 `notification.md`로 이동한다. 일정과 예약, 알림의 연동 정책은 `docs/design-docs/schedule-and-notification.md`로 이동한다.

## 미확정 사항

- 공개 범위 값의 정확한 종류와 일정 유형 값의 관계가 완전히 정의되어 있지 않다.
- 색상 라벨의 선택지와 의미가 정의되어 있지 않다.
- 참석 인원과 참석자 목록의 관계가 정의되어 있지 않다.
- 회의실 예약과 연결된 일정의 수정 및 취소 정책 세부 범위는 `docs/design-docs/schedule-and-notification.md`의 미확정 사항으로 관리한다.
