# 조직도 Product Spec

## 기능 목적

사용자가 조직 구조와 직원 기본 정보를 확인하고, 직원의 소속과 계정 상태 및 근무 상태를 파악할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 조직도를 제공한다.
- 직원 정보를 표시한다.
- 직원 정보에는 프로필 사진, 이름, 직책, 소속팀, 사내 내선번호, 회사 이메일, 계정 상태, 근무 상태를 포함한다.
- 계정 상태는 활성 또는 비활성으로 표시한다.
- 근무 상태는 업무 중, 회의 중, 외근 중, 휴가 중, 오프라인으로 표시한다.
- 조직도에서 직원을 선택하면 해당 직원의 기본 정보를 표시한다.

### 제외한 요구사항

- 직원 등록, 수정, 삭제는 `admin.md`로 이동한다.
- 본인 개인정보 수정은 `my-page.md`로 이동한다.
- 조직도에서 직원과의 채팅 연결 기능은 현재 범위에서 제외한다.
- 직원과의 채팅 연결은 추후 메신저 기능 정의 시 검토한다.

## 화면 구성 요소

- 조직도 영역
- 직원 정보 영역
  - 프로필 사진
  - 이름
  - 직책
  - 소속팀
  - 사내 내선번호
  - 회사 이메일
  - 계정 상태
  - 근무 상태

## 데이터 흐름

1. 조직도 화면에서 팀 계층 정보를 조회한다.
2. 사용자가 조직을 선택하면 해당 조직에 소속된 직원 목록을 표시한다.
3. 사용자가 직원을 선택하면 해당 직원의 기본 정보를 표시한다.
4. 직원의 직책, 소속팀, 사내 내선번호, 회사 이메일, 계정 상태, 근무 상태를 함께 표시한다.
5. 계정 상태와 근무 상태는 서로 구분하여 표시한다.

## 확정된 기능 결정

- 조직은 팀 계층 구조로 표현한다.
- 직원은 하나의 소속 팀과 하나의 직급을 가진다.
- 조직도 조회는 하위 조직까지 표현할 수 있어야 한다.
- 조직도에서 직원 선택 시 직원 기본 정보만 표시한다.
- 직원과의 채팅 연결 기능은 현재 범위에서 제외하며 추후 메신저 기능 정의 시 검토한다.
- 직원의 계정 활성 여부와 근무 상태는 서로 다른 정보로 분리하여 관리한다.
- 계정 상태는 활성 또는 비활성으로 구분한다.
- 근무 상태는 업무 중, 회의 중, 외근 중, 휴가 중, 오프라인으로 구분한다.
- `phone_number`는 사내 내선번호를 저장하는 필드로 사용한다.

## 확정된 기능 결정의 근거

- `ui-ux-spec.md` 1-5에서 조직도와 직원 정보 표시 항목을 정의한다.
- `database.md`의 `users`는 직원 기본 정보와 팀, 직급 연결을 정의한다.
- `database.md`의 `teams`와 `teams_closure`는 팀 계층 구조와 하위 조직 조회 목적을 설명한다.
- `database.md`의 `positions`는 직급 정보를 정의한다.
- 계정 활성 여부와 근무 상태는 의미와 변경 기준이 다르므로 별도 정보로 관리한다.
- 계정 상태는 해당 직원 계정의 서비스 이용 가능 여부를 나타낸다.
- 근무 상태는 직원의 현재 업무 상태를 나타낸다.
- 조직도의 목적은 조직 및 직원 정보 조회이므로 채팅 연결 기능은 현재 범위에서 제외한다.
- 현재 데이터 구조에서는 `phone_number`를 사내 내선번호로 사용한다.

## Scope Decision

- In Scope: 조직도 표시, 직원 기본 정보 조회, 직원 계정 상태 표시, 직원 근무 상태 표시.
- Out of Scope: 직원 관리 CRUD, 본인 정보 수정, 직원과의 채팅 연결.
- Moved: 직원 관리 기능은 `admin.md`, 본인 정보 수정은 `my-page.md`, 직원과의 채팅 연결은 추후 메신저 관련 Product Spec으로 이동한다.
