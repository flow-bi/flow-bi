# 조직도 Product Spec

## 기능 목적

사용자가 조직 구조와 직원 기본 정보를 확인하고, 직원의 소속과 상태를 파악할 수 있도록 한다.

## 상세 요구사항

### 포함한 요구사항

- 조직도를 제공한다.
- 직원 정보를 표시한다.
- 직원 정보에는 프로필 사진, 이름, 직책, 소속팀, 사내 번호, 회사 이메일, 활성화 정보를 포함한다.
- 활성화 정보 예시는 업무중, 휴가중, 외근중 등이다.

### 제외한 요구사항

- 직원 등록, 수정, 삭제는 `admin.md`로 이동한다.
- 본인 개인정보 수정은 `my-page.md`로 이동한다.
- 사람 클릭 시 채팅 연결은 확장 항목으로만 기록한다.

## 화면 구성 요소

- 조직도 영역
- 직원 정보 영역
  - 프로필 사진
  - 이름
  - 직책
  - 소속팀
  - 사내 번호
  - 회사 이메일
  - 활성화 정보

## 데이터 흐름

1. 조직도 화면에서 팀 계층 정보를 조회한다.
2. 선택된 조직 또는 직원 기준으로 직원 정보를 표시한다.
3. 직원의 직책, 소속팀, 연락처, 활성 상태를 함께 표시한다.

## 확정된 기능 결정

- 조직은 팀 계층 구조로 표현한다.
- 직원은 하나의 소속 팀과 하나의 직급을 가진다.
- 조직도 조회는 하위 조직까지 표현할 수 있어야 한다.

## 확정된 기능 결정의 근거

- `ui-ux-spec.md` 1-5에서 조직도와 직원 정보 표시 항목을 정의한다.
- `database.md`의 `users`는 직원 기본 정보와 팀, 직급 연결을 정의한다.
- `database.md`의 `teams`와 `teams_closure`는 팀 계층 구조와 하위 조직 조회 목적을 설명한다.
- `database.md`의 `positions`는 직급 정보를 정의한다.

## Scope Decision

- In Scope: 조직도 표시, 직원 기본 정보 조회, 직원 활성 상태 표시.
- Out of Scope: 직원 관리 CRUD, 본인 정보 수정, 채팅 연결.
- Moved: 직원 관리 기능은 `admin.md`, 본인 정보 수정은 `my-page.md`, AI/채팅 기능은 `ai-assistant.md`로 이동한다.

## Reference Mapping

- `docs/references/ui-ux-spec.md`: 1-5 조직도
- `docs/references/database.md`: `users`, `teams`, `teams_closure`, `positions`

## 미확정 사항

- 조직도에서 사람 클릭 시 채팅 연결은 확장으로만 언급되어 있다.
- 활성화 정보 값의 전체 목록이 정의되어 있지 않다.
- 사내 번호가 `phone_number`와 동일한지 별도 필드인지 명확하지 않다.
