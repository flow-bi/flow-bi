# layout.md

**대응 도메인**: 없음 (프론트 전용, 여러 feature를 감싸는 공통 셸)
**대응 화면**: 화면/기능 설계서에 명시적 스펙 없음 — 화면 간 이동을 위해 이번에 새로 정의
**관련 문서**: `header.md`, `DESIGN.md` 3번(간격/레이아웃), `FRONTEND.md` 6번(라우팅)

## 배경

화면/기능 설계서는 9개 화면(로그인/대시보드/AI채팅/캘린더/조직도/회의실/관리/마이페이지/테마)을 각각 정의했지만, 화면 사이를 이동하는 **공통 셸(헤더+사이드바)** 자체는 스펙이 없었다. 로그인 이후 모든 화면이 이 셸 안에 렌더링된다.

## 레이아웃 구조 (신규 확정)

```
┌─────────────────────────────────────────┐

│           │                             │

```

## 사이드바 네비게이션 항목 (MVP1 확정 화면 기준)

`product-specs/index.md`에서 확정한 MVP1 화면 목록을 그대로 따른다. **AI 채팅(1-3)은 MVP1 스코프에 없으므로 사이드바에 포함하지 않는다.**

| 순서 | 메뉴 | 이동 경로 | 실기능 여부 |
|---|---|---|---|
| 1 | 대시보드 | `/dashboard` | 일부 실기능 (`dashboard.md`) |
| 2 | 캘린더 | `/schedule` | 실기능 |
| 3 | 조직도 | `/org-chart` | 실기능 |
| 4 | 회의실 | `/meeting-rooms` | 실기능 |
| 5 | 마이페이지 | `/my-page` | 실기능 |
| 6 | 관리 | `/admin` | 디자인만, 사이드바에는 노출하되 진입 시 화면 전체가 비활성 톤(`DESIGN.md` "관리 탭 시각적 구분") |
| 7 | 설정 | `/setting` | 디자인만, 사이드바에는 노출하되 진입 시 화면 전체가 비활성 톤(`DESIGN.md` "관리 탭 시각적 구분") |

- **테마 선택**은 별도 사이드바 메뉴가 아니라, 사이드바 하단 또는 헤더 우측의 아이콘 버튼(설정 아이콘)으로 노출한다. 상시 전환할 일이 적은 기능이라 메인 메뉴와 동급으로 두지 않는다.
- 현재 위치한 메뉴는 `DESIGN.md`의 테마 `accent` 색상으로 활성 표시(좌측 4px 바 또는 배경 `accent-soft`).

## 반응형 (NFR-006)

- 데스크톱: 사이드바 상시 노출(240px 고정)
- 모바일: 사이드바는 기본 숨김, 헤더의 햄버거 아이콘으로 오버레이 형태로 열림 (별도 아이콘/트리거는 `header.md` 갱신 필요 — 아래 참고)

## header.md와의 연결 (갱신 필요 사항)

`header.md`에는 햄버거 메뉴 아이콘이 아직 없다. 모바일 대응을 위해 좌측 상단(또는 기업명 옆)에 사이드바 토글 버튼을 추가해야 한다. → 이번 작업에서 `header.md`도 함께 갱신하는 것을 권장.

## 라우팅 가드

`FRONTEND.md` 6번의 인증 가드가 이 레이아웃 셸 전체를 감싼다. 비로그인 상태로 위 경로 중 하나에 접근하면 `/login`으로 리다이렉트하고, 로그인 성공 후에는 원래 가려던 경로로 되돌아간다(redirect-back).

## 남은 미정 사항

- 사이드바에 아이콘(그래픽)을 쓸지, 텍스트만 쓸지 — 아이콘 세트 미정
- 접힘/펼침(collapse) 기능 필요 여부 — MVP1에서는 불필요하다고 가정, 화면이 좁아지면 나중에 검토
