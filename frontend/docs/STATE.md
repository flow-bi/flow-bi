# STATE.md

## State Rules

- 서버에서 받아오는 데이터는 서버 상태로 관리한다.
- 여러 화면에서 공유되는 정보만 전역 상태로 관리한다.
- 단일 화면에서만 사용하는 입력값은 지역 상태로 관리한다.
- 모달 열림 여부, 드로어 열림 여부는 화면 단위 상태로 관리한다.
- 인증 사용자 정보와 테마 정보는 전역 상태로 관리한다.

---

## Global State

전역 상태 후보는 다음과 같다.

- 현재 로그인 사용자
- 인증 토큰
- 사용자 권한
- 선택된 테마
- 사이드바 열림 여부

---

## Local State

### 로그인

- 사번 입력값
- 비밀번호 입력값
- 로그인 요청 상태
- 로그인 오류 메시지

### 캘린더

- 현재 보기 방식
- 선택한 날짜
- 선택한 일정
- 일정 등록 모달 상태
- 일정 상세 모달 상태
- 일정 유형
- 선택한 팀
- 선택한 프로젝트
- 선택한 참석자

### 회의실

- 선택한 날짜
- 선택한 시간대
- 검색 조건
- 선택한 회의실
- 예약 패널 상태
- 예약 폼 입력값

### AI 채팅

- 메시지 목록
- 현재 입력 메시지
- 전송 상태
- 확인 카드 상태
- 선택한 단축어

---

## Server State

서버 상태 후보는 다음과 같다.

- 사용자 목록
- 현재 로그인 사용자 정보
- 팀 목록
- 프로젝트 목록
- 일정 목록
- 일정 상세 정보
- 회의실 목록
- 회의실 예약 현황
- 조직도 데이터
- AI 채팅 응답
