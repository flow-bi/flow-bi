# ARCHITECTURE.md

## 목적

이 문서는 AI Groupware 프론트엔드의 구조와 설계 기준을 기록한다.

프론트엔드의 폴더 구조, 레이어 역할, 데이터 흐름, 컴포넌트 분리 기준을 설명한다.

---

## 기본 원칙

- 화면 중심으로 기능을 구성한다.
- 페이지는 라우팅 단위로 관리한다.
- 페이지 전용 컴포넌트, 타입, mock, 유틸은 해당 페이지 폴더 내부에 둔다.
- 여러 화면에서 재사용하는 요소는 공용 폴더에 둔다.
- 서버 상태와 클라이언트 UI 상태를 구분한다.
- 백엔드 API와 DB 구조를 프론트에서 임의로 변경하지 않는다.

---
