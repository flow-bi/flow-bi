## Main UI Flows

### 로그인

1. 사번과 비밀번호를 입력한다.
2. 로그인 요청을 보낸다.
3. 성공 시 대시보드로 이동한다.
4. 실패 시 오류 메시지를 표시한다.

### 캘린더

1. 기본 화면은 월간 캘린더이다.
2. 월간/주간/일간 보기를 선택할 수 있다.
3. 날짜 클릭 시 해당 날짜 일정을 표시한다.
4. 일정 클릭 시 상세 모달을 연다.
5. 일정 추가 버튼 클릭 시 등록 모달을 연다.
6. 일정 유형은 개인/팀/프로젝트 중 선택한다.

### 회의실

1. 회의실 목록을 조회한다.
2. 날짜, 시간대, 수용 인원, 상태로 검색한다.
3. 예약 버튼 클릭 시 예약 패널을 연다.
4. 예약 성공 시 예약 현황을 갱신한다.

### AI 채팅

1. 사용자가 메시지를 입력한다.
2. 사용자 메시지를 표시한다.
3. AI 응답을 표시한다.
4. 일정 등록, 회의실 예약 등 데이터 변경 요청은 확인 UI를 거친다.

## Phase 1 Priority

로그인 화면 구현 이후 1차 핵심 화면은 다음 순서로 작업한다.

1. 대시보드
2. 회의실
3. 일정관리/캘린더

작업 범위는 한 번에 하나의 화면으로 제한한다.
