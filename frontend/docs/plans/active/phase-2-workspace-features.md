# Phase 2 Workspace Features Plan

## 목적

Phase 1 핵심 업무 화면 구현 이후, 사용자 정보와 조직 구조를 확인할 수 있는 조직도, 마이페이지, 테마 설정 화면을 구현한다.

---

## 작업 순서

1. 조직도
2. 마이페이지
3. 테마 설정

---

## 공통 작업 기준

각 화면은 다음 기준을 만족해야 한다.

- 라우팅이 연결되어야 한다.
- 공통 레이아웃 안에서 표시되어야 한다.
- mock 데이터를 사용해 화면을 구성할 수 있어야 한다.
- Loading, Empty, Error 상태를 고려해야 한다.
- 주요 사용자 상호작용이 동작해야 한다.
- 작업 완료 후 `progress.md`에 기록해야 한다.

---

## 1. 조직도

### 포함 범위

- 부서/팀 구조 표시
- 직원 목록 표시
- 직원 검색
- 직원 상세 정보 패널 또는 모달
- 팀별 필터링

### 제외 범위

- 실제 직원 API 연동
- 조직 변경 기능
- 권한별 조직도 노출 제어

### 완료 기준

- `/organization` 경로에서 조직도 화면이 표시된다.
- mock 조직/직원 데이터가 표시된다.
- 직원 검색이 동작한다.
- 직원 클릭 시 상세 정보가 표시된다.

---

## 2. 마이페이지

### 포함 범위

- 내 프로필 정보 표시
- 부서/직급/이메일/연락처 표시
- 비밀번호 변경 이동 버튼
- 내 일정 요약
- 내 회의실 예약 요약

### 제외 범위

- 실제 프로필 수정 API 연동
- 프로필 이미지 업로드
- 비밀번호 변경 실제 처리

### 완료 기준

- `/mypage` 경로에서 마이페이지가 표시된다.
- mock 사용자 정보가 표시된다.
- 내 일정/예약 요약 정보가 표시된다.
- 비밀번호 변경 버튼 클릭 시 `/change-password`로 이동한다.

---

## 3. 테마 설정

### 포함 범위

- 테마 색상 선택 UI
- 설정 미리보기 영역

### 제외 범위

- 서버 저장
- 사용자별 테마 영구 저장 API 연동

### 완료 기준

- `/settings/theme` 경로에서 테마 설정 화면이 표시된다.
- 테마 선택 UI가 표시된다.
- 선택한 테마가 화면에 반영된다.
