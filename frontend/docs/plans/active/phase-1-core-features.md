# Phase 1 Core Features Plan

## 목적

로그인 화면 구현 이후, 1차 핵심 업무 화면인 대시보드, 회의실, 일정관리/캘린더를 순서대로 구현한다.

---

## 작업 순서

1. 대시보드
2. 회의실
3. 일정관리/캘린더

---

## 공통 작업 기준

각 화면은 다음 기준을 만족해야 한다.

- 라우팅이 연결되어야 한다.
- 공통 레이아웃 안에서 표시되어야 한다.
- mock 데이터를 사용해 화면을 구성할 수 있어야 한다.
- Loading, Empty, Error 상태를 고려해야 한다.(화면상 표시하지않는다.)
- 주요 사용자 상호작용이 동작해야 한다.
- 작업 완료 후 `progress.md`에 기록해야 한다.

---

## 1. 대시보드

### 포함 범위

- AI 오늘 일정 요약 카드
- 오늘의 일정 목록
- 주간 일정 요약
- 팀원 상태 영역

### 제외 범위

- 실제 AI 모델 연동
- 실시간 팀원 상태 연동

### 완료 기준

- `/dashboard` 경로에서 화면이 표시된다.
- mock 데이터 기반으로 오늘 일정, 주간 일정, 팀원 상태가 표시된다.
- 데이터 없음 상태가 표시된다.

---

## 2. 회의실

### 포함 범위

- 회의실 목록
- 회의실 예약 현황
- 날짜/시간/인원/상태 검색 조건
- 예약 패널 또는 모달

### 제외 범위

- 실제 예약 확정 API 연동
- 예약 중복 검증의 최종 처리

### 완료 기준

- `/rooms` 경로에서 화면이 표시된다.
- 회의실 목록과 예약 현황이 표시된다.
- 예약 버튼 클릭 시 예약 UI가 열린다.

---

## 3. 일정관리/캘린더

### 포함 범위

- 월간 캘린더
- 주간/일간 보기 전환
- 날짜별 일정 표시
- 일정 상세 모달
- 일정 등록 모달
- 개인/팀/프로젝트 일정 유형 선택

### 제외 범위

- 실제 일정 등록 API 연동
- 반복 일정
- 외부 캘린더 연동

### 완료 기준

- `/calendar` 경로에서 캘린더가 표시된다.
- mock 일정 데이터가 캘린더에 표시된다.
- 날짜 클릭 시 일정 목록이 표시된다.
- 일정 등록 모달에서 일정 유형을 선택할 수 있다.
